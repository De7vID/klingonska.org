About the Klingonska Akademien Website

This is a collection of info on the guiding principles of the design for this site.

  • Text in page titles and headings (<title> and <hN>) should not end in period.
  • HTML should be:
    • Lower case (<html> not <HTML>).
    • Empty tags should end in />.
    • All tags should be properly terminated (no <p> tag without a </p> etc).
    • Attribute values should be quoted (height="500" not height=500).
    • Attributes are allowed to have no value (pubdate not the redundant pubdate="pubdate").
    • Translations from Klingon should be marked with <i class="transl">...</i>.
    • The <html> tag should have lang attribute specified (e.g. <html lang="en">).
    • Klingon should be marked with
      • Language tlh is used for Klingon text. When Klingon is mixed with other text this usually means using that the language goes on a <b> tag (e.g. <b lang="tlh">tlhIngan Hol</b>). Longer texts (like e.g. Qo'noS QonoS) are less commonly set in bold, and the language attribute is more likely to go on a <p> or <div> wrapping all the paragraphs of text. If the whole page is set in Klingon <html lang="tlh"> might be used for that page.
      • Class affix is used for content of one or more prefixes or suffixes without any root word. It disables word-wrapping for this text – this is to prevent a lone hyphen to wind up last on a line, and the suffix following first on the next line – something which occasionally happens, at least with Google Chrome v8.0 (seen July 2011). E.g. <b lang="tlh" class="affix">-moH</b>.
      • Class sic is used for content supposedly in Klingon, but which does not adhere to correct Klingon spelling (this includes, for example, quoted typos in the Errors in Okrandian Sources article). It is used to note known errors, and may as such be used for also for errors which still renders the word with graphotactically correct spelling. E.g. <b lang="tlh" class="sic">gan</b>.

Page Titles

Page titles look like this:

<title>PAGE TITLE [ &ndash; PROJECT TITLE ] &ndash; Klingonska Akademien</title>

The “&ndash; PROJECT TITLE” part is optional and only appear on certain subpages (e.g. Klingon Pocket Dictionary or Klingonsk Ordbok). The first part of the title (everything up to the first &ndash;) is extracted and used in the page breadcrumb navigation. If there is a PROJECT TITLE defined, then that is used as the page title in the index.html file.

Page titles are used by Google to determine title in the search result presentation. See also: Webmaster Central: Changing your site’s title and description in search results.

Troublesome Unicode Characters

Most unicode is unproblematic, but there are some exceptions with those characters that in some way control the flow of text on the page.

Soft hyphen (&#173; or &shy;) – USED. Indicates a line-breaking point. I line break may be inserted at this point, and if it is, a hyphen is added. Quite common, e.g. to indicate line break points in the Klingon phrases on this web page.

Hard hyphen (&#8208; or “-”) – USED. Indicate a hyphen, after which a line break may, or may not, be inserted. Used in all places where a hyphen is required, e.g. when writing isolated Klingon affixes like yI- and -laH, or in some English words like non-commissioned or names Nielsen-Brown.

Non-breaking hyphen (&#8209;) – NOT USED. Always displays a hyphen, keeps the entire word together in one unit when broken across lines. Could be handy, when writing ISO dates (e.g. YYYY-MM-DD) or unattached Klingon affixes (e.g. HI-), however, this character cannot be found using most browsers search-on-the-page function (ctrl-f), so on this site other means are used to ensure that those words are not being broken.

Minus sign (&minus; or &#8722;) – USED. This is only used in mathematics. E.g. 4+3−1=6.

Should be used for page update times, as well as times elsewhere (e.g. the publication dates listed in the Archive of Okrandian Canon). Uses WHATWG’s HTML5 <time> element.
